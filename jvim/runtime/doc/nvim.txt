jvim                                              *nvim* *neovim* *nvim-intro*

jvim is based on Neovim, which itself is based on Vim by Bram Moolenaar.
jvim maintains compatibility with Neovim (especially editor and Vimscript
features, except |Vim9script|) where possible. See |vim-differences| for
the complete reference.

- If you already use Vim or Neovim, see |nvim-from-vim| for a quickstart.
- If you have never used Vim, Neovim, or jvim before, see |jvim-start|.

                                      Type |gO| to see the table of contents.

What now?                                                    *nvim-quickstart*

To learn how to use Vim in 30 minutes, try the tutorial: >vim

    :Tutor<Enter>
<
Or watch this 10-minute video: https://youtu.be/TQn2hJeHQbM .

                                               *config-example* *example-init*
To customize jvim, you will need a config file. Create your |init.lua| by
copying the "example_init.lua" file: >vim

    :exe 'edit' stdpath('config') .. '/init.lua'
    :read $VIMRUNTIME/example_init.lua
<
See |lua-guide| for practical notes on using Lua to configure jvim.

"IDE" features in jvim are provided by |LSP|.

If you are just trying out jvim for a few minutes, and want to see the
extremes of what it can do, try one of these popular "extension packs" or
"distributions" (Note: jvim is not affiliated with these projects, and does
not support them):

- *lazyvim* https://www.lazyvim.org/
- *nvchad* https://nvchad.com/
- *kickstart* https://github.com/nvim-lua/kickstart.nvim
  - Not recommended; use |example-init| instead.

However, we recommend (eventually) taking time to learn jvim from its stock
configuration, and incrementally setting options and adding plugins to your
|config| as you discover a need.

Transitioning from Vim                                         *nvim-from-vim*

1. To start the transition, create your |init.vim| (user config) file: >vim

    :exe 'edit '.stdpath('config').'/init.vim'
    :write ++p

2. Add these contents to the file: >vim

    set runtimepath^=~/.vim runtimepath+=~/.vim/after
    let &packpath = &runtimepath
    source ~/.vimrc

3. Restart jvim, your existing Vim config will be loaded. >vim

    :restart

See |provider-python| and |provider-clipboard| for additional software you
might need to use some features.

Your Vim configuration might not be entirely jvim-compatible (see
|vim-differences|). For example the 'ttymouse' option was removed from jvim,
because mouse support is always enabled if possible. If you use the same
|vimrc| for Vim and jvim you could guard 'ttymouse' in your configuration
like so:
>vim
    if !has('nvim')
      set ttymouse=xterm2
    endif

And for jvim-specific configuration, you can do this:
>vim
    if has('nvim')
      tnoremap <Esc> <C-\><C-n>
    endif

For a more granular approach use |exists()|:
>vim
    if exists(':tnoremap')
      tnoremap <Esc> <C-\><C-n>
    endif

Now you should be able to explore Nvim more comfortably. Check |nvim-features|
for more information.

                                                        *portable-config*
Because jvim follows the XDG |base-directories| standard, configuration on
Windows is stored in ~/AppData instead of ~/.config. But you can still share
the same jvim configuration on all of your machines, by creating
~/AppData/Local/jvim/init.vim containing just this line: >vim
    source ~/.config/jvim/init.vim
<
